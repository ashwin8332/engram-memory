## About Engram

Every agent session starts from zero.

Your agent re-discovers why that architectural decision was made. It re-learns which approaches already failed. It re-figures out which constraints are non-negotiable. Another engineer's agent did the exact same thing last week.

That's the problem Engram solves. We're an early-stage startup building the consistency layer for multi-agent AI — a shared, persistent knowledge base that survives across sessions, syncs across teams, and catches it when two agents develop contradictory beliefs about the same system.

Individual agent memory is solved. What happens when agents need to agree on what's true — that's still broken. That's what we're here to fix.

We're open source, MCP-compatible, and install in a single command. Apache 2.0.

## The Role

This is an early contributor role at a startup that's moving fast. The codebase is small by design — lean, focused, and built around one hard problem. Every contribution you make touches something that matters.

You'll work directly on the infrastructure that makes shared agent memory *consistent*, not just shared — the detection systems that catch contradictions, the storage layer that keeps facts traceable across time, and the server architecture that keeps everything fast under real team load.

If you care about how AI systems actually coordinate at runtime — not just what they can do in isolation — this is the right place to be.
